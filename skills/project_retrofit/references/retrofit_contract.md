# Retrofit contract

`project_retrofit` applies when the repository already contains meaningful legacy material that must remain visible while a Generic Project Harness control layer is planned and, when safe, added.

The skill must:

1. inventory the repository before any overlay
2. classify the current control-layer state
3. stop and redirect to `project_evolution` when conflicting or partial control files are already present
4. preserve `results/`, `outputs/`, `reports/`, and `notebooks/` in place by default
5. treat README modification as an explicitly approved action only
6. set the post-overlay handoff to `stage_identification`
